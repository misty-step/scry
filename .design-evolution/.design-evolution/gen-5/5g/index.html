\\n What is the time complexity of finding an element in a balanced binary search tree?\\n
\\n\\n
\\n \\n \\n \\n \\n
\\n
\\n
\\n
\\n
\\n\\n \\n
\\n
\\n
\\n \\n
\\n \\n Correct\\n
\\n
\\n\\n
\\n
\\n
Your Answer
\\n
O(log n)
\\n
\\n\\n
\\n
Correct Answer
\\n
O(log n)
\\n
\\n\\n
\\n
Explanation
\\n
\\n In a balanced binary search tree (like AVL or Red-Black tree), each comparison allows you to eliminate half of the remaining nodes. This divide-and-conquer approach results in logarithmic time complexity O(log n), where n is the number of nodes. This is significantly more efficient than linear search O(n) in an unsorted array.\\n
